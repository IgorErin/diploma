\subsection{Рантайм}

tokio представляет два асинхронных рантайма:

\begin{itemize}
    \item \verb|current_thread| представляет собой однопоточный рантайм, то есть для исполнения асихронных задач используется один поток
    \item \verb|multi_thread| представялет собой многопоточный рантайм, где каждый т. н. воркер, сущность, ответственная за исполнение асихронных задач представленна отдельно и исполняется на отдельном thread'e из blocking pool
\end{itemize}

Команда tatlin.backup использует исключительно current_thread рантайм, потому в данной работе будет рассматриваться шедулер только упомянутого выше рантайма.

\subsection{Воркер}

Воркер представляет сущность ответсвенную за выполнение асихронных задач. Он имеет локальную SPMC очередь задач, к которую помещаются задачи, продуцированные в ходе исполенния задачи в данный момент обрабатываемой воркером. Только сам воркер может добавлять задачи в собственную локальную очередь.

\subsection{Inject очередь}

Inject очередь --- это глобальная MPMC очередь задач доступная всем воркерам. Пополняется она в случае

\begin{itemize}
    \item Шедулинга задачи вне контекста конкретного воркера, например, из потока блокирующего тредпулла.
    \item при переполеннии локальной очереди задач воркера, в этом случае, воркер переносит половину локальных задач в глобальную очередь.
\end{itemize}
